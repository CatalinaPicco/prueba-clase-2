Pruebas en GIT

Clase 1 sobre repositorios y comandos

.gitignore es un archivo para poner las excepciones que no queremos subir al repo o trackear.
.gitkeep es para agregar carpetas.
rm -f nombre del archivo : comando para borrar.
git log : tira el historial.
git log --oneline

Para subir a la nube:
git push origin master: el origin es el repo remoto en el cual estamos trabajando y el "master" es la rama de desarrollo que quieran mandar cambios para arriba
git pull origin master : es para traer desde la nube a repo local.

Clase 2 sobre repositorios y comandos

Versionar es hacer un snapshot de los archivos que nos interesan conservar.

git reset HEAD + nombre de archivo del cual hicimos add y queremos sacar del stage, porque no nos interesa commitear. git checkout -- + nombre del archivo que quiero eliminar. cambios de nuevo
